# AI 教案项目开发流程硬性规则

用户于 2026-06-17 在 1-AI教案项目中明确本规则，所有后续开发必须遵守。

## 规则清单（按顺序）

1. **语雀原文抓取先于开发**
   - 任何新 W 阶段启动前必须先把语雀 dhluml 对应专题的原始文档抓到本地（落盘 .md）
   - 后续开发（拆解/重写/合并）必须基于已落盘的真实文档，不允许凭空生成
   - 抓取工具：`fetch_yuque.mjs`（密码 ghkq，需 Puppeteer headful + Chrome CDP 9222）

2. **每子阶段完成后做端到端验收 + PRD 规格检视**
   - 端到端验收 = 启动 dev server + 用 `scripts/snap.mjs` 截屏 + 控制台无错 + 关键路由可访问
   - PRD 规格检视 = 逐条对照子阶段 PRD 中声明的 FR/AC 是否真的实现
   - **若发现较大偏差或虚假验收风险立即停下来找用户确认**，不允许"凑合通过"

3. **下一阶段开始前必须做两件事**
   - (a) 根据总 PRD 文档制定该阶段**验收标准**和**开发计划**
   - (b) 把验收标准 + 开发计划**单独总结成文**（一个 md 文件），给出**审计意见**
   - **开发执行前必须先闭环所有审计意见**，直到无新增致命或重大规格偏差/风险

4. **真实数据验收**
   - 禁止用 mock/估算数据假装通过验收
   - 验收数据源必须能从 lessons.json / dist / 实际页 DOM 中读出来
   - **验收不通过 → 打回开发计划阶段重新思考并执行**，不允许多次重复提交

5. **自动化优先**
   - 验收/审计/开发尽量由我（Claude）自动完成
   - 人类只处理高风险流程（流程变更/重大偏差/无法判定的问题）

## 触发场景

- 任何 W 阶段 (W3/W4/W5/...) 启动
- 任何子阶段 (W3.A/W3.B/W3.C) 收尾
- 任何工具/页面/课程模块上线前

## 实施清单（每次阶段必走）

```
1. 抓取语雀原文 → 落盘 .md → 文件清单 + 字数确认
2. 写本阶段开发计划 (md) + 验收标准 (md) + 审计意见 (md)
3. 等用户确认无致命/重大偏差后执行开发
4. 开发完毕跑 Puppeteer snap 截屏 + 0 console error
5. PRD 规格逐条对照，发现偏差 → 打回计划阶段
6. 验收通过 → 写本阶段 W*_ACCEPTANCE.md → 提交推送
```

**Why:** 用户已经多次反馈"页面空白/虚假验收"问题，明确规则避免重复。
**How to apply:** 任何 1-AI教案项目相关 task，先检查当前所在 W 阶段，按规则清单走 6 步。
